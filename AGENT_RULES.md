# EYE-D Agent Rules

이 파일은 EYE-D 프로젝트에서 AI 에이전트(코딩 어시스턴트)가 코드를 작성하거나 시스템을 조작할 때 반드시 지켜야 하는 규칙을 정의합니다. 에이전트는 작업을 시작하기 전 항상 이 규칙을 참조해야 합니다.

## 1. 언어 정책 (Language)
- 에이전트의 모든 답변, 코드 설명, 문서 작성은 반드시 **한국어(Korean)**로 진행합니다.

## 2. Git 커밋 규칙 (Git Commits)
- **커밋 메시지 언어**: `git commit` 시 작성하는 커밋 메시지 본문은 반드시 **한국어**로 작성합니다. (예: `feat(server): OSNet 기반 유사도 검색 로직 추가`)
- **커밋 시점 통제**: 에이전트는 코드를 수정하고 저장(`git add` 등)할 수는 있으나, **사용자가 명시적으로 "커밋해 줘"라고 요청하기 전까지 절대로 `git commit`을 자동으로 수행하지 않습니다.**

## 3. 포트 및 테스트 환경 가이드 (Testing & Environment)
- **UI 접속 기준**: 프론트엔드 UI 검증을 안내할 때, 별도의 개발 포트(3000번 등)가 아닌 **FastAPI 서버의 기본 포트인 `http://localhost:8000` 접속을 기본으로 안내**합니다. (FastAPI의 `main.py`가 프론트엔드 빌드 및 서빙을 통합해서 처리하기 때문입니다.)
- **Mock 데이터 주입**: E2E 테스트가 필요할 경우 `server/tools/bulk_mock_sender.py`를 활용하며, 실행할 때마다 시드가 랜덤하게 적용되어 새로운 데이터가 들어가는 것을 전제로 안내합니다.

## 4. 문서화 최신화 (Documentation)
- 프로젝트 설계나 엔드포인트, 테스트 방법이 변경되면 즉시 아래의 문서들에 변경 사항을 반영해야 합니다:
  - `docs/TESTING.md`: E2E 테스트 및 UI 검증 가이드
  - `docs/server-plan.md`: 전체 서버 구현 Phase 체크리스트
  - `server/docs/server_analysis.md`: 서버 아키텍처 및 API 엔드포인트 스펙
