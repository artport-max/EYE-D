# EYE-D Control Center UI Requirements

## 1. 개요
Jetson Orin Nano 기반의 실내 지능형 관제 시스템(EYE-D)을 위한 고성능 대시보드 인터페이스 요구사항입니다. Edge 디바이스의 성능을 모니터링하고 실시간으로 인원을 추적 및 Re-ID(재식별)하는 기능을 시각화하는 데 중점을 둡니다.

## 2. 디자인 시스템
- **테마**: 다크 테마 (Dark-first design)
- **주요 색상**: 
  - Primary: Blue-600 (#2563eb)
  - Success/Detection: Emerald-500 (#10b981)
  - Critical/Alert: Red-500 (#ef4444)
  - Background: Black/Deep Grey (#0a0a0c, #0d0d0f)
- **타이포그래피**: 
  - 기본: Inter (Sans-serif)
  - 데이터/시간: JetBrains Mono (Monospace)
- **효과**: 유리 질감(Backdrop-blur), 스캔라인 시뮬레이션, 바운딩 박스 애니메이션.

## 3. 화면 구성 및 레이아웃

### 3.1 GNB (Global Navigation Bar) - 좌측
- 16px 너비의 슬림형 수직 네비게이션.
- 아이콘 중심의 메뉴 (대시보드, 통계, 관리, 검색, 설정, 알림).
- 활성화된 상태를 시각적으로 강조.

### 3.2 헤더 (Header) - 상단
- 시스템 명칭: "EYE-D CONTROL CENTER"
- 디바이스 태그: "Jetson Orin Nano" (배지 형태)
- **실시간 시스템 메트릭**: 
  - GPU 사용률 (%)
  - CPU 사용률 (%)
  - GPU 온도 (°C)
- 실시간 현재 시간 표시 (24시간 포맷).

### 3.3 메인 대시보드 (Main View) - 중앙
- **4분할 비디오 그리드 (4-Split Grid)**:
  - 4개의 카메라 피드를 동일한 비율로 배치.
  - 선택된 피드에 하이라이트(파란 테두리) 표시.
  - 각 피드 상단에 카메라 이름 및 상태(LIVE/RECORDING) 표시.
  - 마우스 호버 시 전체화면 옵션 제공.
  - **시뮬레이션 오버레이**: 탐지된 인원 위에 실시간 바운딩 박스 및 라벨 표시.

- **실시간 Re-ID 로그 피드 (Real-time Log Feed)**:
  - 하단에 위치한 스트리밍 로그 영역.
  - 로그 구성:
    - 썸네일 (탐지된 인원의 얼굴/전신 크롭)
    - Person ID: 고유 식별 번호
    - 동선(Trajectory): 출발지 카메라 → 목적지 카메라
    - 신뢰도(Confidence): 매칭 확률 (%)
    - 발생 시각: 초 단위 타임스탬프

### 3.4 사이드바 (Analytics Sidebar) - 우측
- **인원 통계 차트**: 
  - 시간별 인원 밀도 (Area Chart).
  - 유입 인원 총계 및 점유율(Occupancy) 지표.
- **카메라 제어 패널**:
  - 선택된 카메라의 상세 제어 (객체 탐지, 추적, 마스크 체크, 비식별화 On/Off).
  - 스트림 사양 정보: FPS, 해상도, 비트레이트, 지연시간(Latency).
  - Re-ID 데이터베이스 검색 버튼.

## 4. 서브 화면: 인물 히스토리 조회 (Person History Lookup)
- **개요**: 특정 Person ID를 기준으로 과거 이동 경로와 탐지 이력을 시뮬레이션 및 데이터 형태로 제공.
- **레이아웃**:
  - **프로필 상단 (Profile Header)**: 
    - 대상 인물의 베스트 스냅샷.
    - Person ID, 첫 탐지 시각, 마지막 탐지 위치, 누적 탐지 횟수.
    - 위험 등급 표시 (Normal, Suspicious, Restricted).
  - **수직 타임라인 (Vertical Timeline)**:
    - 시간 역순으로 인물의 이동 경로 표기.
    - [탐지 시각] - [카메라 위치] - [신뢰도] - [개별 비디오 피드] 형태로 카드 배치.
    - 각 카드 내부에 독립적인 스캔라인 및 바운딩 박스 오버레이 시뮬레이션 포함.
    - 이동 소요 시간(Travel Time)을 노드 사이에 표기.
  - **이동 경로 브리핑**: 텍스트 형태의 동선 요약 (예: "Entrance(14:00) -> Lobby(14:15) -> Office(14:30)").
- **상호작용**:
  - 타임라인 카드 클릭 시 해당 시점의 녹화 영상 재생 기능 (시뮬레이션).
  - 필터: 특정 날짜, 특정 카메라 구역별 필터링.
  - 액션: 화이트리스트 등록, 블랙리스트(경보) 등록, 동선 데이터 CSV 추출.

## 5. 기능적 세부 요구사항
- **실시간성**: 로그 데이터는 5초 주기로 자동 갱신되며 리스트 상단에 추가됨.
- **상호작용**: 그리드 레이아웃의 카메라를 클릭하면 우측 제어 패널이 해당 카메라 컨텍스트로 전환됨.
- **애니메이션**: 로그 추가 시 슬라이드 효과, 시스템 메트릭 변동 시 부드러운 상태 전환.
- **반응형**: 데스크탑 화면에 최적화하되, 컨테이너 크기에 따라 그리드 레이아웃이 유연하게 조절됨.
